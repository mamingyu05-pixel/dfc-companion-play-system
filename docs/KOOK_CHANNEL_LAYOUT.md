# May猫饼 KOOK 频道结构建议

目标：让客户知道去哪咨询，让陪玩知道去哪报名，让管理员知道去哪审核。

## 推荐分组

### 欢迎大厅

- `新人接待`
  - 用途：新用户进服后的第一入口。
  - 建议：只放欢迎语、规则、客服入口。
- `入会申请`
  - 用途：客户或陪玩申请加入社群。
  - 建议：陪玩入驻不要开放自助注册，统一提示联系客服考核。
- `公告`
  - 用途：活动、维护、优惠说明。

### 客户服务

- `客服接待`
  - 用途：客户咨询、AI 客服、人工接管。
  - 建议：客户可见可发言；AI 客服先回复，复杂问题转人工。
- `订单需求`
  - 用途：客户发找陪玩需求。
  - 第一阶段可以先不单独建，用 `客服接待` 承接。

### 试音与下单

- `试音等候室`
  - 用途：客户先进入，客服再拉陪玩试音。
  - 建议：不要让所有陪玩自由进入，避免打扰客户。
- `人工派单`
  - 用途：客服发布需求，陪玩报名。
  - 建议：客户不可见；陪玩和管理员可见。

### 陪玩专区

- `陪玩公告`
  - 用途：接单规则、处罚规则、活动奖励。
- `接单大厅`
  - 用途：陪玩查看可接需求。
  - 如果已使用 `人工派单`，可以暂时不建。
- `陪玩反馈`
  - 用途：陪玩反馈订单问题、提现问题。

### 运营后台

- `充值审核`
  - 用途：充值提醒、截图核对、到账确认。
  - 权限：仅管理员。
- `提现审核`
  - 用途：提现申请、人工打款确认。
  - 权限：仅管理员。
- `投诉处理`
  - 用途：退款、投诉、争议订单。
  - 权限：仅管理员。
- `管理提醒`
  - 用途：系统异常、Bot 失败日志、管理员待办。
  - 权限：仅管理员。

## AI 客服策略

AI 客服适合处理：

- 问候和引导。
- 找陪玩需求收集。
- 预算不确定时的解释。
- 试音流程说明。
- 充值流程说明。
- 优惠码、邀请码说明。
- 陪玩入驻流程说明。

AI 客服不能直接处理：

- 加余额。
- 退款。
- 提现完成确认。
- 封号。
- 投诉结论。
- 修改订单金额。

这些动作必须由管理员后台操作，并留下管理员日志和钱包流水。

## 运营前测试话术

在 KOOK 私聊或 `客服接待` 频道测试：

```text
你好
我想下单
没有预算可以吗
我想找三角洲陪玩，晚上玩2小时，想先试音
充值没到账怎么办
我想做陪玩
```

期望效果：

- AI 能连续理解上下文。
- 不会每句话都转人工。
- 充值、退款、提现等高风险动作仍提示人工确认。

## 发布频道欢迎消息

部署后运行：

```bash
cd /opt/companion-play-system
sudo git pull origin main
sudo docker compose build api-server
sudo docker compose run --rm api-server node /app/scripts/post-kook-welcome.js
```

运行后，到 KOOK 里把各频道的说明消息手动置顶。
