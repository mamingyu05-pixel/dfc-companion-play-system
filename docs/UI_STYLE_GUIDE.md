# DFC UI 风格指南

## 设计目标

DFC 的界面必须同时满足三个要求：

1. 客户端有电竞氛围，但不能像游戏活动页一样花哨。
2. 陪玩端操作直接，手机和 PC 都能快速接单、看收益。
3. 管理后台高密度、低干扰，适合长时间处理审核、订单和财务。

## 视觉关键词

- 电竞感。
- 暗黑风。
- 高级感。
- 简洁。
- 信任感强。
- 运营效率高。

## 使用规则

- 页面背景只用 `dfc.bg`。
- 卡片、表格、弹窗用 `dfc.surface` 或 `dfc.elevated`。
- 蓝紫色只做按钮、焦点、选中态和少量边缘高亮。
- 金色只用于 VIP、推荐陪玩、优先展示。
- 红色只用于失败、警告、封禁、拒绝、危险操作。
- 不使用大面积渐变背景，不使用装饰性光球。

## 字体与字号

- 默认字体：Inter / system UI。
- 页面标题：24px / 32px。
- 区块标题：18px / 28px。
- 卡片标题：16px / 24px。
- 正文：14px / 22px。
- 辅助文字：12px / 18px。

## 圆角与间距

- 卡片圆角：8px。
- 按钮/输入框圆角：6px。
- 弹窗圆角：8px。
- 移动端页面边距：16px。
- PC 页面边距：24px。
- 管理后台表格行高：44px 到 52px。

## 阴影与发光

- 默认不使用大阴影。
- 只在按钮 focus、推荐陪玩、当前选中订单上使用轻微蓝紫发光。
- 发光不能影响文字可读性。

## 交互状态

- Hover：背景轻微变亮。
- Active：按钮下压或颜色加深。
- Focus：使用 `shadow-dfc-focus`。
- Disabled：降低透明度，鼠标不可点击。
- Loading：按钮显示加载状态，禁止重复提交。

## 可访问性要求

- 文字与背景对比度必须足够。
- 表单错误必须有文字提示，不能只靠颜色。
- 危险操作必须二次确认。
- 所有移动端按钮高度不低于 40px。
