# DFC 响应式布局规范

## 断点

| 场景 | 宽度 |
| --- | --- |
| Mobile | `< 768px` |
| Tablet | `768px - 1023px` |
| Desktop | `>= 1024px` |
| Wide Admin | `>= 1440px` |

## Customer Portal

移动端优先。

- 主要导航使用底部 Tab。
- 陪玩列表单列卡片。
- 筛选器使用底部抽屉。
- 下单页采用单列步骤式布局。
- 充值页必须把“人工审核说明”放在提交前。

PC：

- 陪玩列表使用 3 到 4 列网格。
- 筛选器固定在左侧或顶部。
- 下单页使用左右布局：左侧表单，右侧价格摘要。

## Companion Portal

手机和 PC 同等重要。

移动端：

- 工作台首屏显示当前订单、待接订单、收益。
- 接单按钮固定在订单卡片底部。
- 提现申请使用单列表单。

PC：

- 左侧导航。
- 工作台使用指标卡 + 订单列表。
- 我的订单和收益明细使用表格。

## Admin Portal

PC 优先，但移动端必须可用。

PC：

- 左侧固定导航。
- 顶部工具栏。
- 表格为主要信息载体。
- 筛选条件保持可见。

移动端：

- 表格降级为卡片列表。
- 审核操作放在详情抽屉底部。
- 危险操作必须二次确认。

## 布局尺寸

- 页面最大宽度：
  - Customer：1200px。
  - Companion：1180px。
  - Admin：无固定最大宽度，内容区最小 1024px 体验最佳。
- 移动端左右内边距：16px。
- PC 内容区内边距：24px。
- 卡片间距：12px 到 16px。
- 管理后台密集表格间距：8px 到 12px。

## 触控规范

- 移动端按钮高度不低于 40px。
- 列表项可点击区域不低于 44px。
- 底部固定操作栏高度 56px 到 64px。
- 弹窗在手机上优先使用底部抽屉。
