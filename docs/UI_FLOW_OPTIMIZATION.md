# DFC 用户流程优化建议

## 下单流程

目标：减少客户从陪玩列表到支付下单的流失。

建议：

1. 陪玩卡片上直接展示核心决策信息：
   - 在线状态。
   - 三角洲段位。
   - 擅长模式。
   - 单价。
   - 语音偏好。
   - 是否支持试音。
2. 陪玩详情页只保留下单需要的信息。
3. 下单页不要让用户手填金额。
4. 余额不足时直接给出“去充值”按钮。
5. 下单确认前展示：
   - 单价。
   - 时长。
   - 总价。
   - 当前余额。
   - 下单后余额。
6. 如果客户没有选好陪玩，提供“平台代选”入口。
7. 平台代选必须让客户填写语音偏好、模式、时长和备注，便于管理员人工挑人。

## 充值流程

目标：让用户清楚理解第一阶段是人工审核。

建议：

1. 充值页顶部明确说明：
   - 当前为人工审核。
   - 提交截图后由管理员审核。
   - 审核通过后余额增加。
2. 上传截图前展示转账步骤。
3. 提交后显示审核状态。
4. 被拒绝时显示拒绝原因。
5. 充值记录保留截图和备注。

## 陪玩卡片

目标：提高点击率和下单转化。

建议：

1. 在线陪玩优先排序。
2. 推荐陪玩使用金色小标识，不大面积铺金色。
3. 价格清晰展示，避免用户点进去才看到。
4. 使用状态 Badge：
   - 在线。
   - 忙碌。
   - 离线。
5. 移动端卡片必须单手可读，按钮在底部。
6. 卡片必须展示语音风格，例如教学型、指挥型、安静型。
7. 支持试音的陪玩显示“试音”入口，但试音不等于订单开始。

## 管理后台

目标：提高管理员审核和派单效率。

建议：

1. 数据看板首屏展示待处理事项。
2. 充值审核、提现审核、派单、投诉单独作为快捷入口。
3. 表格必须支持状态筛选和搜索。
4. 审核弹窗必须保留备注。
5. 危险操作必须二次确认。
6. 财务相关操作必须显示操作后影响。

## 陪玩端

目标：减少陪玩漏单和提现疑问。

建议：

1. 工作台首屏展示可接订单和当前订单。
2. 收益页明确区分：
   - 可提现收益。
   - 待结算收益。
3. 提现申请页显示预计处理方式：人工打款。
4. 订单完成后提示收益状态变化。
