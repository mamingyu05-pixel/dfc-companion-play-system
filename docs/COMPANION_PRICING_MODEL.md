# May猫饼 陪玩定价模型

## 原则

当前不要设置全平台统一单价。陪玩价格应该由管理员根据实力、服务能力和供需情况单独设置。

订单金额以后端计算为准：

```text
订单金额 = 陪玩每小时价格 * 服务时长
```

平台人工挑人价格使用：

```env
PLATFORM_MATCH_UNIT_PRICE=
```

如果这个值为空，客户下单页会提示“客服报价后确认”，避免前端显示错误价格。

## 建议陪玩等级

| 等级 | 定位 | 建议价格范围 |
| --- | --- | ---: |
| 试用陪玩 | 新入驻、需考核、适合低预算娱乐单 | 待运营确认 |
| 标准陪玩 | 普通娱乐、稳定沟通、基础上分 | 待运营确认 |
| 高段陪玩 | 高段位、强操作、可带上分 | 待运营确认 |
| 教学陪玩 | 复盘、教学、路线/意识讲解 | 待运营确认 |
| 明星陪玩 | 稀缺、强人设、高复购 | 待运营确认 |

不要在系统里直接写死价格范围，运营初期需要用真实成交数据回调。

## 影响价格的因素

- 游戏和模式热度。
- 段位、胜率、KD、历史评价。
- 是否可语音、声音表现、沟通稳定性。
- 是否支持试音。
- 是否支持教学/复盘。
- 黄金时段、节假日、热门活动。
- 是否为客户指定陪玩，还是平台人工匹配。

## 后台操作建议

1. 管理员创建陪玩时必须手动填写 `每小时价格`。
2. 初期每周复盘一次成交和流失：
   - 点击多但下单少：可能价格过高或资料不够可信。
   - 下单多但投诉多：可能陪玩等级标得过高。
   - 高复购陪玩：可逐步涨价。
3. 平台人工挑人价格不要长期固定，建议等运营稳定后再设置。

## 后续可开发

建议后续增加字段：

```text
companionTier: TRIAL / STANDARD / HIGH_RANK / COACH / STAR
ratingScore
completedOrderCount
repeatCustomerCount
```

再做自动建议价，但第一阶段仍由管理员最终确认价格。
